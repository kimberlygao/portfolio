Navigating Carnegie Mellon University’s Student Information Online (SIO) remains a challenge for the student body for scheduling classes, viewing finances, requesting transcripts and many other administrative tasks. The following redesign aims to improve the user experience of Carnegie Mellon’s SIO, specifically for class scheduling. After multiple rounds of prototyping and user testing, the new SIO design centers around being able to easily view and add classes to minimize the user’s time while maximizing functionality.

01 Problem

Current SIO

Browse Courses

View Courses

Looking at the current SIO, I observed many issues I wanted to address in my redesign.

The main issues included:

  • cognitive overload across all pages
  • lack of accessibility in color constrast of text
  • cannot browse all courses without filtering by at least one category or search

Other issues I observed but did not make a priority to address included:

  • no differentiation between undergrad and graduate courses
  • waitlisting and course reservation transparency
  • cannot view schedule while browsing courses

02 Rapid Prototyping

I began by creating 3 initial rapid prototypes in 20 minutes on Figma.

P1. Daily Schedule with Sidebar Control
P2. List Schedule and Separate Control Page
P3. Table Schedule and Tabs for Features

I conducted a quick user study on each of these prototypes and received the following feedback:

  • P1 offers traditional schedule view design pattern which is hard to visualize mentally
  • P2 and P3 require user to remember the schedule plan when searching for a new schedule
  • P3 dropdown filter offers recognition over recall
  • P3 produces cognitive overload with all the description

Overall, the P1 showed better schedule functionality with a simplified sidebar control on the left side and calendar view on the right. P3's dropdown filter eases search and browse functionality for users.

03 Hi-fi Prototype

I defined a few design goals which guided my design choices:

  • fast functionality for busy student life
  • full course catalog with simple browsing
  • quick links to popular resources

Moving forward, I decided to take the best visual and most convenient aspects of the 3 prototypes. I wanted to include more symbols and dropdown menus (instead of input boxes). I altered the button placement for more familiar locations and simplified the search feature with the option for advanced search.

04 Heuristic Evalution

After learning about Nielsen's usability engineering method, Heuristic Evaluation, I closely examined my design to find potential usability problems.

I identifed 3 heuristics to improve upon:

  1. Recognition rather than recall

    The home page contains unnecessary information for the user to access after logging in. The design should be focused on the essentials for schedule planning and browsing in this context.

  2. Consistency and standards

    The navigation menu is ambiguous on “Academics”. Users might not know how to navigate to the course catalog in contrast to the option for scheduling.

  3. Aesthetics and minimalist design

    the home page contains unnecessary information for the user to access after logging in. The design should be focused on the essentials for schedule planning and browsing in this context.

05 Development V1

Based on my heuristic evaluation, I made some design changes as I began development with HTML + CSS + JS in building an initial version of my site

My goal was to have the home page, course catalog, and course details minus the functionality.

I faced a lot of development challenges with trial and error for different solutions. The biggest challenge was spacing and layout for all pages. I discovered css grid as an alternative to flexbox but had trouble making the page responsive. I couldn't solve the issue with spacing between the CMU logo square and line. Another part that caused some trouble was the form on the details page for a course since it was a form.

06 Development V2

In the final stretch of the project, I needed to build out the schedule page as well as implement the functionality to add a class to the schedule. This aspect was very JavaScript heavy in working with objects, session storage, and manipulating the DOM with jQuery.

I also received feedback from peers about my design in which I removed the tan accent color and replaced it with all gray.
